# Launch Checklist

Use this checklist before every real participant launch.

## Study setup

- consent text is final and approved
- instructions match the actual study flow
- compensation and duration are accurate
- completion code or return URL is correct
- pilot data files are cleared or archived if needed

## Prolific

- study URL is the correct deployed URL
- `PROLIFIC_PID`, `STUDY_ID`, and `SESSION_ID` are captured correctly
- eligibility filters are correct
- payment settings are correct
- screen-out and return policies are documented

## Technical checks

- study works on desktop
- study works on mobile if mobile participation is allowed
- participant cannot bypass consent
- data files are written as expected
- completion flow matches the Prolific setup

## Archiving

- the exact launched study version is archived
- launch date and settings are documented
- any study-specific deviations from the template are recorded
